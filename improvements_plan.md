# План доработок системы Overtime

## 1. Оптимизация размера Docker-образа бэкенда (CPU-only PyTorch)
* **Статус:** ✅ **Выполнено**
* **Цель:** Сократить размер Docker-образа бэкенда с ~10 ГБ до ~2 ГБ, а также уменьшить потребление диска и ОЗУ на сервере.
* **Описание:** Поскольку проект развернут на CPU-сервере без дискретных видеокарт Nvidia CUDA, библиотека PyTorch с CUDA-драйверами (532 МБ скачиваемого архива и гигабайты распакованных библиотек CUDA) избыточна.
* **Реализация:**
  * В [Dockerfile.base](file:///home/nik/Dev/overtime-prod/backend/Dockerfile.base) настроена установка CPU-версий `torch` и `torchaudio` напрямую из официального репозитория PyTorch (`https://download.pytorch.org/whl/cpu`).
  * Для предотвращения лишней нагрузки удаляется библиотека `triton` (нужная только для GPU-вычислений).

## 2. Предварительная загрузка ML-моделей в Docker-образ
* **Статус:** ✅ **Выполнено**
* **Цель:** Исключить долгое скачивание моделей (Whisper и Gazeta) из интернета при первом запуске контейнера или первом голосовом запросе пользователя.
* **Описание:** Модели Whisper и RuT5 скачиваются и кэшируются в Docker Volume при первом запуске контейнера. Ранее использовалась gazeta. В настоящее время используется rut5-base для суммаризации текстов.
* **Реализация:**
  * Создан скрипт [download_models.py](file:///home/nik/Dev/overtime-prod/backend/scripts/download_models.py), скачивающий Whisper "base" и T5 "rut5_base_sum_gazeta" в папки `/app/.cache/whisper` и `/app/models/`.
  * В [entrypoint.sh](file:///home/nik/Dev/overtime-prod/backend/scripts/entrypoint.sh) настроен асинхронный фоновый запуск скрипта скачивания при первом старте контейнера, что позволяет веб-серверу Uvicorn запускаться мгновенно без ожидания загрузки моделей. После завершения скачивания создается маркер-файл `/app/models/.models_ready`.

## 4. Интеграция производственного календаря (РК) и умная корректировка времени переработок
* **Цель:** Исключить ошибки при расчете часов и автоматизировать отправку напоминаний о запуске переработки с учетом выходных и праздников Республики Казахстан.
* **Описание:**
  1. **Производственный календарь Казахстана**:
     * Использовать интеграцию с API `isdayoff.ru?cc=kz` для автоматического получения праздничных дней.
     * Создать локальную таблицу `holidays` в базе данных для хранения исключений (праздники в будни и рабочие выходные из-за переносов), чтобы не зависеть от стабильности внешних API.
  2. **Умная корректировка времени старта (Start Time Truncation)**:
     * Если переработка начата сотрудником в рабочий день раньше **19:00**, бэкенд при расчете часов автоматически сдвигает время старта на **19:00**.
     * В выходные и праздничные дни переработка учитывается полностью с момента запуска (без сдвигов).
  3. **Уведомления-напоминания в Telegram**:
     * Ежедневная задача в `JobQueue` бота, запускающаяся в **19:00** в рабочие дни Казахстана.
     * Бот отправляет сообщение-напоминание о необходимости запустить таймер только тем сотрудникам, у которых привязан Telegram, нет активной сессии переработки и в профиле включена опция `overtime_reminders_enabled` (для предотвращения спама).
