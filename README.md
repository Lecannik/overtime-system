# 🕒 OvertimePro (Система учета переработок)

**OvertimePro** — это современная корпоративная система для автоматизации учета, согласования и аналитики внеурочных работ (переработок). Система позволяет эффективно управлять ресурсами компании, прозрачно отслеживать затраты времени и формировать отчетность для принятия управленческих решений.

---

## ✨ Основные возможности

*   **👥 Ролевая модель доступа**: 
    *   **Сотрудник**: Создание заявок на переработку, просмотр личной статистики.
    *   **Менеджер**: Согласование заявок своей команды, аналитика по курируемым проектам.
    *   **Начальник отдела**: Контроль бюджета времени всего отдела, утверждение заявок после менеджеров.
    *   **Администратор**: Полный доступ к управлению пользователями, отделами, проектами и глобальной аналитике.
*   **📊 Аналитическая панель**: Визуализация трудозатрат в разрезе проектов, отделов и конкретных сотрудников за любые периоды.
*   **📂 Экспорт отчетов**: Выгрузка детальных данных в Excel с автоматическим расчетом итогов и профессиональным форматированием.
*   **🔐 Безопасность**: Двухфакторная аутентификация (OTP) через Microsoft Office Integration и контроль прав на уровне API (Security at Depth).
*   **⚖️ Умный расчет**: Автоматический подсчет часов с учетом правил округления на стороне базы данных.

## 🚀 Быстрый старт

### Способ 1: Использование Docker (Рекомендуется)

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/your-repo/overtime-system.git
    cd overtime-system
    ```
2.  Создайте файл `.env` на основе примера:
    ```bash
    cp .env.example .env
    ```
3.  Запустите проект:
    ```bash
    docker-compose up --build
    ```
    Миграции базы данных применятся автоматически при запуске бэкенд-контейнера. Система будет доступна по адресу `http://localhost:8090` (через Nginx Gateway).

### 🔑 Создание администратора по умолчанию

Для создания первоначальной учетной записи администратора выполните команду:
```bash
docker-compose exec backend python scripts/create_admin.py
```
или при локальном запуске:
```bash
python scripts/create_admin.py
```
Скрипт автоматически создаст пользователя с email `admin@example.com`, сгенерирует безопасный случайный временный пароль и установит флаг смены пароля при первом входе.

### Способ 2: Локальный запуск (для разработки)

#### Backend
1. Перейдите в папку `backend`:
   ```bash
   cd backend
   python -m venv venv
   source venv/bin/activate  # Или venv\Scripts\activate для Windows
   pip install -r requirements.txt
   ```
2. Запустите сервер:
   ```bash
   uvicorn app.main:app --reload
   ```

#### Frontend
1. Перейдите в папку `frontend`:
   ```bash
   cd frontend
   npm install
   ```
2. Запустите dev-сервер:
   ```bash
   npm run dev
   ```

## ⚙️ Настройка окружения (`.env`)

Для корректной работы системы необходимо настроить следующие переменные в файле `.env`:

*   `POSTGRES_...` — параметры подключения к базе данных.
*   `SECRET_KEY` — секретный ключ для создания JWT-токенов.
*   `MS_...` — параметры интеграции с почтовым сервисом Microsoft для отправки уведомлений и кодов OTP.

Полный список доступен в файле [`.env.example`](./.env.example).

## 🌐 Развертывание (Production Deployment)

Продакшн-версия системы развертывается по принципу инфраструктуры без исходного кода на целевом сервере (Source-code-free Deployment):

1. **Сборка и доставка**:
   * Сборка Docker-образов бэкенда (`lecannik/overtime-backend`) и фронтенда (`lecannik/overtime-frontend`) выполняется автоматически в CI-пайплайне GitHub Actions.
   * Образы тегируются уникальным SHA коммита (`IMAGE_TAG=${{ github.sha }}`) и отправляются в реестр DockerHub.
2. **Деплой на сервере**:
   * GitHub Actions Runner (`git-runner`) с помощью протокола `scp` копирует на сервер деплоя файлы конфигурации: `docker-compose.prod.yml` и `nginx.conf`.
   * По протоколу `ssh` на сервере выполняется обновление контейнеров:
     ```bash
     export IMAGE_TAG=<commit_sha>
     docker compose -f docker-compose.prod.yml pull
     docker compose -f docker-compose.prod.yml up -d
     ```
3. **Запуск административных скриптов на продакшене**:
   Так как исходный код встроен в Docker-образы, служебные скрипты запускаются непосредственно внутри контейнеров хост-командами:
   * Создание учетной записи администратора:
     ```bash
     docker compose -f docker-compose.prod.yml exec backend python scripts/create_admin.py
     ```
   * Накатывание миграций базы данных (выполняется автоматически при старте бэкенда или вручную):
     ```bash
     docker compose -f docker-compose.prod.yml exec backend alembic upgrade head
     ```
